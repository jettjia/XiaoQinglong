# XiaoQinglong
核心定位
小青龙 Agent OS 是一款企业级智能体运行框架，专注于为 AI Agent 提供完整的后端支撑体系，实现多渠道接入、多模型路由编排、安全沙箱执行的一体化解决方案。

核心特点
1. 多渠道统一接入
支持 Web / API / 飞书 / 钉钉 / 微信 等多种渠道
统一 Gateway 层处理鉴权、限流、消息标准化
渠道可灵活插拔，扩展新渠道成本低
2. 多模型智能路由
按对话角色自动选择最优模型：
default - 主对话模型
rewrite - Query 改写优化
skill - 技能执行模型
summarize - 内容摘要模型
支持 OpenAI / Anthropic / Google / Azure 等主流 LLM
降低模型调用成本，提升响应质量
3. 强大的编排能力 (Orchestrator)
可视化编排 Agent 执行流水线
支持推理阶段 (Reasoning) → 工具调用 (Tools) → 审批节点 (Approval) → 响应生成
配置化替代硬编码，快速构建复杂业务流程
4. 多协议并存
HTTP - RESTful API，通用性强
gRPC - 高性能内部通信
MCP (Model Context Protocol) - 标准化工具扩展协议
5. 技能中心 (Skill Center)
内置 Skills 市场，支持自定义技能开发
支持 MCP Server 接入外部工具
A2A (Agent-to-Agent) 协议实现多 Agent 协作
6. 知识库增强 (RAG)
无缝对接外部知识检索 API
支持 Top-K 召回与重排序 (Rerank)
提升 Agent 回答的准确性与专业性
7. 安全沙箱执行
Docker 隔离环境运行高风险工具
可配置环境变量、超时时间、工作目录
保护宿主系统安全
8. 企业级架构
层次	说明
API 接入层	HTTP/gRPC/MCP/WebSocket 多协议适配
Gateway 层	验签鉴权、限流控制、消息标准化、Agent 路由
核心服务层	会话中心、技能中心、编排中心、工具服务、知识库服务
基础设施层	数据库 (PostgreSQL)、缓存、日志、配置中心
9. 多格式响应
text - 纯文本
markdown - 富文本
json - 结构化数据
a2ui - 前端组件化渲染
image/audio/video - 多媒体生成
multipart - 混合响应

10.  完善的运维支持
流式响应 (SSE)
指数退避重试机制
完整执行追踪 (Trace) - 模型/延迟/Token/工具调用详情
多环境配置 (debug / test / release)
技术栈
层级	技术
后端	Go + Gin + GORM + PostgreSQL
前端	React + TypeScript + Vite + TailwindCSS
架构	DDD (Domain-Driven Design) 分层
协议	HTTP / gRPC / MCP / WebSocket
运行	Docker 沙箱隔离

11. 适用场景
智能客服 - 多渠道接入，统一回复
企业知识助手 - RAG 增强的内部知识检索
流程自动化 - 编排复杂业务流程，多工具协作
Agent 开发平台 - 为第三方提供 Agent 能力输出
任意系统的互联互通

## Runner 特性

### 多模型路由
支持按角色选择不同模型，优化成本和性能:

| 角色        | 说明           | 用途            |
| ----------- | -------------- | --------------- |
| `default`   | 默认主对话模型 | 主对话流程      |
| `rewrite`   | Query 改写模型 | 用户 Query 优化 |
| `skill`     | Skill 执行模型 | 技能执行        |
| `summarize` | 总结模型       | 内容摘要        |

```json
"options": {
    "routing": {
        "default_model": "default",
        "rewrite_prompt": "优化以下用户Query...",
        "summarize_prompt": "请总结以下内容..."
    }
}
```

### 响应格式 (response_schema)
支持多种输出格式，可通过 `response_schema.type` 指定:

| 类型                | 说明            | 用途            |
| ------------------- | --------------- | --------------- |
| `text`              | 纯文本          | 简单文本回复    |
| `markdown`          | Markdown 格式   | 富文本回复      |
| `json`              | JSON 结构化     | structured data |
| `a2ui`              | A2UI 组件化格式 | 前端组件渲染    |
| `image/audio/video` | 多媒体          | 媒体生成        |
| `multipart`         | 多格式混合      | 复合响应        |

```json
"response_schema": {
    "type": "a2ui",
    "schema": { ... }
}
```

### 流式响应
启用 SSE 流式输出:

```json
"options": {
    "stream": true
}
```

### 重试机制
指数退避重试策略:

```json
"options": {
    "retry": {
        "max_attempts": 3,
        "initial_delay_ms": 1000,
        "max_delay_ms": 10000,
        "backoff_multiplier": 2.0,
        "retryable_errors": ["timeout", "rate_limit"]
    }
}
```

### 限流控制
```json
"options": {
    "max_tool_calls": 10,
    "max_a2a_calls": 5,
    "max_iterations": 10,
    "max_total_tokens": 8000
}
```

### 执行 Metadata
响应包含完整执行详情:

| 字段                | 说明                                       |
| ------------------- | ------------------------------------------ |
| `model`             | 使用的模型                                 |
| `latency_ms`        | 总延迟(毫秒)                               |
| `prompt_tokens`     | Prompt token 消耗                          |
| `completion_tokens` | Completion token 消耗                      |
| `tool_calls_count`  | 工具调用次数                               |
| `a2a_calls_count`   | A2A 调用次数                               |
| `skill_calls_count` | Skill 调用次数                             |
| `iterations`        | 迭代次数                                   |
| `tool_calls_detail` | 工具调用详情(名称/输入/输出/延迟/成功状态) |
